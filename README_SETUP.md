# 해커스 영어 단어 퀴즈 — 설치 · 실행 안내

iOS 스타일 영어 단어 퀴즈 앱입니다. **진짜 구글 로그인 + 기기 간 동기화**를 지원해요.
로그인하면 저장한 단어·학습 기록이 **계정에 저장돼서, 폰·PC 어디서 로그인해도 똑같이** 보입니다.

---

## 무엇이 들어있나

- **3단계** — 1단계(메인 60단어) · 2단계(유의어·반의어 289단어) · ★저장함
- **3가지 모드** — 시험지 · 빈칸 · 매칭 (페이지당 8단어, 즉시 채점)
- **진행률 스크러버** — 바를 끌거나 탭해서 페이지 이동, 완료 페이지는 그라데이션으로 채워짐
- **연속(streak)** — 정답 연속 횟수 + 최고 기록
- **다시 풀기** — 같은 페이지를 보기 새로 섞어 재시도
- **별표 저장(☆) + 오답 자동 저장** — ★저장함 탭에서 저장한 단어만 모아 풀기
- **구글 로그인 + 동기화** — 계정별로 저장·기록이 서버에 보관되어 여러 기기에서 공유

> 로그인하지 않아도 모든 학습 기능은 동작합니다. 이때 기록은 **이 기기(브라우저)** 에만 저장돼요.
> 로그인하면 그때부터 **계정(서버)** 에 저장되어 다른 기기와 동기화됩니다.

---

## 1. 준비물

- **Node.js 18 이상** (https://nodejs.org 에서 설치)

## 2. 실행 (로그인 없이 바로 써보기)

```bash
npm install
npm start
```

브라우저에서 **http://localhost:3000** 접속. 이 상태로도 저장·복습 등 모든 기능이 동작합니다.
(구글 로그인 버튼을 누르면 "설정이 필요하다"는 안내 모달이 떠요. 아래 3번을 마치면 켜집니다.)

---

## 3. 진짜 구글 로그인 켜기

### 3-1. OAuth 클라이언트 ID 발급 (무료)

1. https://console.cloud.google.com 접속 → 프로젝트 생성(또는 선택)
2. 좌측 메뉴 **API 및 서비스 → OAuth 동의 화면**
   - User Type: **외부(External)** 선택 → 앱 이름·이메일 등 기본 정보 입력 후 저장
   - (테스트 단계라면 "테스트 사용자"에 본인 구글 계정을 추가)
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **웹 애플리케이션**
   - **승인된 자바스크립트 원본(Authorized JavaScript origins)** 에 앱 주소를 추가:
     - 로컬 테스트: `http://localhost:3000`
     - 실제 배포: `https://여러분의도메인.com`
   - 만들기 → 표시되는 **클라이언트 ID** 를 복사 (예: `1234567890-abc...apps.googleusercontent.com`)

### 3-2. 서버에 클라이언트 ID 넣고 실행

macOS / Linux:
```bash
GOOGLE_CLIENT_ID="복사한_클라이언트_ID" \
SESSION_SECRET="아무_긴_무작위_문자열" \
npm start
```

Windows (PowerShell):
```powershell
$env:GOOGLE_CLIENT_ID="복사한_클라이언트_ID"
$env:SESSION_SECRET="아무_긴_무작위_문자열"
npm start
```

이제 **http://localhost:3000** 에서 "Google 로그인" 버튼이 실제로 동작합니다.
로그인하면 저장·기록이 계정에 동기화돼요.

> `SESSION_SECRET` 을 고정값으로 주면 서버를 재시작해도 로그인이 유지됩니다.
> 안 주면 보안을 위해 재시작마다 로그아웃돼요(로컬 테스트엔 무방).

## 4. 인터넷에 배포해서 다른 기기와 공유하기

다른 기기에서도 같은 계정으로 동기화하려면 앱이 **공개된 웹 주소(https)** 에 올라가 있어야 합니다.

- 추천: **Render / Railway / Fly.io / 직접 운영하는 서버** 등 Node 앱을 돌릴 수 있는 호스팅
- 배포 시 환경변수 `GOOGLE_CLIENT_ID`, `SESSION_SECRET`, `NODE_ENV=production` 설정
- 그리고 3-1의 **승인된 자바스크립트 원본**에 배포 주소(`https://...`)를 꼭 추가하세요.
- 저장 데이터는 서버의 `.data/users.json` 파일에 보관됩니다. 호스팅에서 파일이 유지되는
  디스크(또는 볼륨)를 쓰거나, 운영 규모가 커지면 `server/store.js` 를 Postgres 등으로 교체하세요.

> ⚠️ GitHub Pages·Netlify 정적 호스팅 등 "파일만 올리는" 방식으로는 동기화가 안 됩니다.
> 로그인 검증과 데이터 저장을 처리할 **이 Node 서버가 함께 떠 있어야** 합니다.

## 폴더 구조

```
server/
  index.js     서버 (정적 파일 제공 + 구글 로그인 검증 + 동기화 API)
  store.js     사용자 데이터 저장소 (JSON 파일; Postgres 등으로 교체 가능)
public/
  index.html   앱 진입점
  app.js       화면 로직 (헤더·단계·진행률 스크러버·저장함·로그인)
  quiz-modes.js  시험지/빈칸/매칭 렌더러 (원본 보존)
  vocab-data.js  단어 데이터 + 헬퍼 (원본 보존)
  vendor/      React (로컬 번들, 외부 CDN 불필요)
```

## 동기화 API (참고)

| 메서드 | 경로 | 설명 |
| --- | --- | --- |
| GET | `/api/config` | 구글 클라이언트 ID·로그인 활성 여부 |
| GET | `/api/me` | 현재 로그인한 사용자 |
| POST | `/api/auth/google` | 구글 ID 토큰 검증 → 세션 발급 |
| POST | `/api/logout` | 로그아웃 |
| GET | `/api/data` | 내 저장·기록 불러오기 (로그인 필요) |
| PUT | `/api/data` | 내 저장·기록 저장 (로그인 필요) |
